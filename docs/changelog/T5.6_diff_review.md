## Новий файл

### `app/plan_runtime/tools.py` — +372 рядки

9 функцій. Всі DB-імпорти lazy. Повністю тестовані без реальної БД.

#### `create_first_plan(user_id) → dict`
```
- user.current_state == "IDLE_ONBOARDED" → else ValueError
- profile.daily_time_slots["DAY"] встановлено → else ValueError
- create_plan(db, user_id, "SHORT", day_time, evening_time=None)
- user.current_state = "ACTIVE" + db.commit()
- activate_plan_side_effects(plan.id, user_id)  ← P1 fix: schedules APScheduler jobs
- return {"status": "ok", "plan_type": "SHORT"}
```

#### `create_followup_plan(user_id, plan_type) → dict`
```
- plan_type: "SHORT" | "MEDIUM" → else ValueError
- user.current_state in {IDLE_FINISHED, IDLE_DROPPED, IDLE_PLAN_ABORTED} → else ValueError
- MEDIUM без evening_slot_collected → return {"status": "needs_evening_time"}
- create_plan(...) → ACTIVE + activate_plan_side_effects(plan.id, user_id)
```

#### `record_evening_time(user_id, hhmm) → dict`
```
- hhmm формат ^\d{2}:\d{2}$ → else ValueError
- profile.daily_time_slots["EVENING"] = hhmm
- profile.evening_slot_collected = True
```

#### `change_day_time(user_id, hhmm) → dict`
```
- update_user_time_slots(db, user, {"DAY": hhmm})
  → recompute_future_steps() перезаписує scheduled_for в БД  ← P1 fix
  → повертає (updated_ids, active_ids)
- reschedule_plan_steps(active_ids)  ← тепер зі правильним часом
- return {"status": "ok", "day_time": hhmm, "rescheduled": N}
```

#### `change_evening_time(user_id, hhmm) → dict`
Аналогічно для EVENING slot.

#### `get_plan_status(user_id) → dict`
```
- {"state": current_state, "plan_active": bool, "days_total": N, "days_completed": N}
```

#### `pause_plan(user_id) → dict`
```
- user.current_state == "ACTIVE" → else ValueError
- delegates to app.plan_pause.pause_plan()
- current_state = "ACTIVE_PAUSED"
```

#### `resume_plan(user_id) → dict`
```
- user.current_state == "ACTIVE_PAUSED" → else ValueError
- delegates to app.plan_pause.resume_plan()
- current_state = "ACTIVE"
```

#### `cancel_plan(user_id) → dict`
```
- user.current_state in {"ACTIVE", "ACTIVE_PAUSED"} → else ValueError
- plan.status = "abandoned", plan.end_date = now
- user.current_state = "IDLE_PLAN_ABORTED"
- cancel_plan_step_jobs(pending/delivered step IDs)
```
