## P1-фікс (попередній PR): міграція `canceled_by_adaptation`

Codex підняв коректний баг:

> **P1 — Preserve canceled semantics when dropping the flag**
> Migration rewrites `canceled_by_adaptation = TRUE` → `expired`, but `expired` = user missed delivery window → counted in `silent_miss_rate`. Migrated cancellations inflate churn metrics for any users who had adapted-away steps.

**Фікс у `migrations/20260530_drop_adaptation_schema.sql`:**

```diff
-SET step_status = 'expired'
+SET step_status = 'canceled'
```

`'canceled'` — вже валідний статус у `plan_guards.py`, `telegram.py`, `scheduler.py`. **Не включений** у `_fetch_eligible_steps` (`["completed", "skipped", "expired"]`), тобто ці кроки коректно виключаються з метрик.
