## Що зараз ламається

`DraftBuilder` читає поля `logic_tags`, `balancing` зі JSON-бібліотеки вправ.
Цих полів більше нема → `KeyError` при кожній спробі створити план.

## Файли які переписуємо

### 1. `app/plan_drafts/service.py` — переписати повністю

Замінити весь файл на одну публічну функцію `create_plan()`:

```python
from datetime import datetime, timezone
from sqlalchemy.orm import Session
from app.db import AIPlan, PlanDraftRecord, PlanDraftStep

def create_plan(
    db: Session,
    user_id: int,
    plan_type: str,            # "SHORT" | "MEDIUM"
    day_time: str,             # "HH:MM"
    evening_time: str | None,  # "HH:MM", required якщо MEDIUM
) -> AIPlan:
    """
    Будує план через PlanBuilderV5, зберігає draft record, одразу фіналізує.
    Повертає активний AIPlan. Без user-facing confirmation step.

    active_days/work_days читаються всередині з user_profile —
    orchestrator не передає їх як параметр.
    """
    from app.plan_drafts.plan_builder_v5 import get_default_builder
    from app.plan_finalization import finalize_plan

    builder = get_default_builder()
    draft_v5 = builder.build(
        plan_type=plan_type,
        user_id=str(user_id),
        day_time=day_time,
        evening_time=evening_time,
    )

    draft_record = _persist_v5_draft(db, user_id, draft_v5)
    db.flush()

    plan = finalize_plan(
        db, user_id, draft_record,
        activation_time_utc=datetime.now(timezone.utc),
    )
    return plan
```

`_persist_v5_draft` — приватна функція яка записує `PlanDraftRecord` + `PlanDraftStep` з `PlanDraftV5`.  
Поля `focus` і `load` в `PlanDraftRecord` = `NULL` (вже `nullable=True`).

**Старі функції** `build_plan_draft`, `persist_plan_draft`, `get_latest_draft`, `delete_latest_draft`, `overwrite_plan_draft` — **видалити**.  
Тести які їх використовують — видалити разом (вони тестують мертвий код).
